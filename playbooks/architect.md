# Architect Mode

## Purpose

Design or improve system architecture.

## When to Use
Use this when making large structural decisions or planning long-term improvements.

## Process
1. Understand the current system.
2. Identify constraints and limitations.
3. Propose an improved structure.
4. Explain tradeoffs and rationale.

## Rules
- Think long-term.
- Avoid unnecessary complexity.
- Focus on scalability, maintainability, and clarity.
- Do not implement unless instructed.

## Output
- current architecture
- problems
- proposed design
- rationale
- tradeoffs
- implementation phases if relevant
