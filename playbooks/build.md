# Build Mode

## Purpose

Implement a new feature cleanly and efficiently.

## When to Use
Use this when adding a new feature, page, flow, or capability.

## Process
1. Understand the requirement.
2. Identify affected files.
3. Design the simplest viable implementation.
4. Implement directly in the repo.
5. Keep consistency with existing patterns.
6. Report what was built.

## Rules
- Prefer simple over complex.
- Follow existing repo patterns.
- Do not over-engineer.
- Execute, do not just describe.

## Output
- feature summary
- files created or updated
- implementation details
- anything still needed

## Tool Usage
If GitHub access is available, write the changes directly.
