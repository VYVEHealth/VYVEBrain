# Refactor Mode

## Purpose

Improve structure without changing functionality.

## When to Use
Use this when the code is messy, inconsistent, or hard to maintain.

## Process
1. Identify messy or inconsistent code.
2. Restructure for clarity.
3. Align with existing repo patterns.
4. Keep changes controlled and safe.

## Rules
- Do not change behavior.
- Avoid large risky rewrites.
- Keep refactors scoped and understandable.

## Output
- what was refactored
- why
- files changed
- anything that needs later follow-up
