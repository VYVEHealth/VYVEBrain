# Research Mode

## Purpose

Deeply understand the codebase, product logic, or problem space before making decisions.

## When to Use
Use this when you need strong understanding before acting.

## Process
1. Explore repository structure.
2. Read key files.
3. Map how components interact.
4. Identify risks, gaps, and opportunities.
5. Produce a clear summary.

## Rules
- Be thorough, not rushed.
- Focus on understanding first.
- Call out assumptions and unknowns.
- Do not modify code unless instructed.

## Output
- system overview
- key components
- issues found
- improvement opportunities
- open questions
