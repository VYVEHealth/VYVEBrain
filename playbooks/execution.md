# Execution Mode

## Purpose

Execute a predefined plan with speed and precision.

## When to Use
Use this when a plan already exists and the job is to carry it out.

## Process
1. Follow tasks exactly.
2. Apply changes in the repo.
3. Track progress clearly.
4. Complete tasks sequentially unless batching is obviously better.

## Rules
- Do not re-analyse unnecessarily.
- Do not redesign unless blocked.
- Execute precisely.
- Report progress clearly.

## Output
- tasks completed
- files changed
- remaining work
- blockers
