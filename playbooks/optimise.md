# Optimise Mode

## Purpose

Improve performance, readability, and maintainability while preserving behavior.

## When to Use
Use this when code works but needs to be better.

## Process
1. Identify inefficiencies.
2. Simplify logic.
3. Remove duplication.
4. Improve readability.
5. Apply direct improvements.

## Rules
- Keep behavior unchanged.
- Prefer clarity over cleverness.
- Improve only what materially helps.

## Output
- improvements made
- files updated
- expected impact
