# Review Mode

## Purpose

Review code for quality, bugs, structure, and practical improvements.

## When to Use
Use this when checking work that already exists.

## Process
1. Inspect relevant files.
2. Identify issues:
   - bugs
   - poor structure
   - inefficiencies
   - inconsistencies
3. Suggest practical improvements.
4. Apply obvious safe fixes if appropriate.

## Rules
- Be critical but useful.
- Prioritise high-impact issues.
- Do not rewrite unnecessarily.

## Output
- issues found
- recommendations
- safe fixes applied
- risks still open
